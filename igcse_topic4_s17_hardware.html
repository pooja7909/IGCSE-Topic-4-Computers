17.1Stored Programs & The Von Neumann Model
▼

Early computers like a calculator could only do one job — they had to be physically rewired to do something different. The idea of a general-purpose computer was developed in the 1930s–40s through the work of Alan Turing and John von Neumann.

SUBJECT VOCABULARY

Von Neumann architecture — a computer system design in which the program is stored in memory with the data. Also called a stored-program computer.

CPU (Central Processing Unit) — the hardware device that carries out all the processing in a computer.

RAM (Random-Access Memory) — the main memory; a temporary store for data and instructions (programs).

Bus — a group of connections (wires) between devices in a computer.

✏️ Key idea: In von Neumann architecture, instructions are stored in memory alongside the data. The CPU fetches these instructions from memory, decodes them and executes them. This is why you can install different programs on the same computer without rewiring it.
17.2Main Memory: RAM, ROM, Cache & Virtual Memory
▼

🔵 RAM — Random Access Memory

RAM is the temporary store the CPU uses for data and instructions currently being processed. It is volatile — contents are lost when power is turned off. RAM is typically measured in GB.

The CPU uses the bus to read (retrieve data from a memory address) and write (send data to a memory address) to RAM. Each memory location has a unique memory address — like a row of letter boxes, each holding 1 byte.

🟢 ROM — Read-Only Memory

ROM is non-volatile — its contents are NOT lost when power is off. It stores firmware: small programs permanently built into the hardware. The contents cannot be altered by ordinary programs.

SUBJECT VOCABULARY

Firmware — software that is permanently stored in ROM (or flash memory) and controls the basic hardware functions of a device. It is rarely updated and is tied directly to the hardware it controls.

Booting (the boot process) — the sequence of steps a computer performs when it is powered on, before the operating system loads.

BIOS (Basic Input/Output System) — the traditional firmware stored in ROM that runs on startup. Replaced in modern computers by UEFI.

UEFI (Unified Extensible Firmware Interface) — the modern replacement for BIOS; faster, supports larger storage, more secure.

🥾 The Boot Process — Step by Step

What happens when you press the power button?
1
Power on — electricity flows to the CPU. RAM is empty. The CPU has no instructions yet.
2
Firmware runs from ROM — the CPU automatically reads the BIOS/UEFI stored in ROM. This is the only memory with contents at this point.
3
POST (Power-On Self-Test) — the firmware checks that essential hardware components (RAM, keyboard, storage) are present and working correctly.
4
Locate the OS — the firmware looks for the operating system on the hard disk (or SSD). It checks a special area called the boot sector.
5
Load the OS into RAM — the OS (e.g. Windows, Linux) is copied from secondary storage into RAM so the CPU can run it.
6
OS takes control — the operating system loads the desktop and the computer is ready to use.
💡 Why is firmware in ROM? Because ROM is non-volatile — even if RAM is wiped or the hard disk fails, the firmware is still there. Without it, the CPU would have nothing to read when it first powers on and the computer could never start.

✏️ Exam tip: "Why is the BIOS stored in ROM?" Answer: Because ROM is non-volatile — the firmware must still be present even after the power has been off. RAM would lose the firmware when power was removed so it cannot be used to store the BIOS.

🟠 Cache Memory

Cache is a small amount of very fast, expensive memory located between the CPU and RAM. In the 1980s CPUs and RAM ran at the same speed, but CPUs became much faster than RAM — creating a bottleneck. Cache solves this by storing frequently used data and instructions so the CPU can access them at its own fast speed without waiting for RAM.

🔍 Cache miss: If the CPU needs data not already in the cache, it must wait for the cache to reload from RAM — this is called a cache miss.

There are three levels of cache: L1 (fastest, smallest, inside each core), L2 (medium, per core), L3 (largest, shared by all cores). The CPU checks L1 first, then L2, then L3. A larger cache = fewer cache misses = faster processing.

🟣 Virtual Memory

When a computer is running the OS and several applications at the same time, RAM can become full. Instead of closing programs, the OS uses a clever trick: it borrows space on the hard disk to act as extra (pretend) memory. This pretend memory is called virtual memory.

🔄 How Virtual Memory Works — Step by Step
1
A process running needs to store data but RAM is full — there is no free space.
2
The OS memory manager identifies the data in RAM that has been least recently used.
3
That data is swapped out — copied from RAM to a reserved area on the hard disk called the swap file (or page file).
4
The freed RAM space is now used for the new data the running process needs.
5
If the swapped-out data is needed again, it is swapped back in from the hard disk to RAM — but something else must be swapped out first.

17.3CPU Components & The Fetch–Decode–Execute Cycle
▼
SUBJECT VOCABULARY

ALU (Arithmetic/Logic Unit) — the part of the CPU that performs calculations and logic operations.

Register — a storage location inside the CPU used to hold an instruction, address, or item of data. Very fast to access.

Control Unit (CU) — the part of the CPU that organises the actions of all other parts; controls the FDE cycle.

Clock — an electronic device that 'ticks' at regular intervals, synchronising all CPU actions. Measured in GHz (1 GHz = 1000 million ticks/second).

Bus width — the number of wires in a bus. Determines how many binary digits (1s and 0s) can be communicated at once.

The three buses connecting CPU, RAM, ROM and I/O devices

Address Bus

Carries memory addresses from CPU to RAM (one direction only). Tells RAM which location to read from/write to.

Data Bus

Carries data values between CPU and memory (bidirectional). Transfers the actual 1s and 0s being read or written.

Control Bus

Carries control signals from the CU (e.g. read, write, clock, interrupt). Coordinates timing of all operations.

🔄 Fetch–Decode–Execute in Detail

Step 1 — FETCH
1
CU places the address of the next instruction (from Program Counter) on the address bus
2
CU sends a read signal on the control bus to RAM
3
RAM finds the data at that address and puts it on the data bus
4
CPU copies the instruction from the data bus into a register
Step 2 — DECODE
1
The Control Unit analyses the contents of the register
2
Sends signals to the relevant parts of the CPU telling them what to do (e.g. "add these numbers", "store this value")
Step 3 — EXECUTE
1
The instruction is carried out by the CPU (e.g. ALU performs a calculation; result stored in accumulator; data written back to memory)
→
The cycle then repeats continuously until the power is turned off
✏️ Exam tip: For FDE questions, always mention the specific bus used at each step (address bus for fetch, data bus to carry instruction, control bus for read/write signals) and the registers involved. These are the mark-scoring details.
17.4Factors Affecting CPU Performance
▼

There are three main factors that affect how fast a CPU performs:

⏱ Clock Speed

Rate at which instructions are processed. Faster clock = more instructions per second. Modern CPUs: ~3 GHz.

Limits: Transistors have a maximum operating rate; high speed generates excessive heat (even liquid nitrogen needed at 9 GHz).

🔢 Processor Cores

More cores = more tasks simultaneously. Dual-core (2), Quad-core (4), Octa-core (8).

Limits: Not all programs can use multiple cores — sequential tasks cannot be parallelised, so a second core gives no benefit.

💾 Cache Size

Larger cache = more frequently used data stored close to CPU = fewer cache misses = CPU waits less for RAM.

L1 (fastest/smallest) → L2 → L3 (slowest/largest, shared).

17.5Secondary Storage: Magnetic, Optical & Solid-State
▼

Secondary storage is any permanent (non-volatile) storage that keeps data when the power is off. It is slower, cheaper, and has much larger capacity than RAM (typically in terabytes).

💡 All secondary storage records data as binary 1s and 0s — but each type uses a different physical mechanism to do so.
🧲 Magnetic (HDD)
How it works: Spinning platters with magnetic coating. North/south poles represent 1s and 0s. A read head floats above the spinning disk (110 km/h) to read/write.

Seek time = arm moves to right track; Latency = right sector rotates under head.

✅ Large capacity · Cheap per GB
❌ Slow · Fragile (moving parts) · Can be noisy
💿 Optical (CD/DVD/Blu-Ray)
How it works: Laser reads pits (less reflective) and lands (more reflective) along a spiral track. Differences in reflection = 1s and 0s.

Writing: laser heats material to create non-reflective pits.

✅ Portable · Cheap per disc
❌ Scratches cause data loss · Slower · Lower capacity than HDD
⚡ Solid-State (SSD/USB)
How it works: NAND flash chips with transistors. Trapped electrons = 0, empty pool = 1. No moving parts.

Flash drives can only be rewritten ~1 million times before failing.

✅ Fastest · Silent · Durable · Small size
❌ More expensive per GB · Limited write cycles

17.6Cloud Storage & Virtualisation ⭐ 6-Mark Topic
▼
⭐ This is frequently a 6-mark exam question. You must give detailed, developed points — not just one-word answers. Each advantage/disadvantage should include a reason why it matters.
SUBJECT VOCABULARY

Cloud storage — secondary storage, often belonging to a third party, that is accessed via a network (usually the Internet). Files stored 'in the cloud' can be accessed from anywhere via an Internet connection.

Virtualisation — any process that hides the true physical nature of a computing resource, usually to simplify the way it is accessed. Cloud storage appears to be a single hard drive, but may actually be spread across hundreds of servers in data centres around the world.

File server — a computer on a network that stores files for other computers to access.

NAS (Network-Attached Storage) — a hard disk connected directly to a network, accessible by all users on that network.

☁️ How Cloud Storage Works

Cloud storage is provided by companies who own large data centres filled with thousands of hard drives and servers. When you save a file to Google Drive or Dropbox, it is uploaded via the Internet to one of these servers. The file might be stored on multiple servers in different locations to ensure it is never lost. You can access it from any device with an Internet connection — it appears as if you have a single, large hard drive, even though the actual storage is distributed across the world (this is virtualisation).

✅ Advantages of Cloud Storage — Exam-Quality Points
1
Access from anywhere, on any device — Files can be accessed from any device with an Internet connection (laptop, phone, tablet). This means you are not tied to one computer and can work remotely. For example, a student can access their homework from school and home.
2
Automatic backup by the provider — The cloud storage company automatically backs up your data across multiple servers. If one server fails, your data is still safe. You do not have to remember to back up manually, reducing the risk of data loss.
3
Easy to share and collaborate — Files can be shared with others by sending a link. Multiple users can work on the same document simultaneously (e.g. Google Docs), which improves collaboration without emailing files back and forth.
4
No need to buy or maintain hardware — The storage is provided by a third party. You do not need to purchase physical hard drives or maintain servers. This reduces cost and technical responsibility, especially for businesses.
5
Scalable storage — Cloud storage can easily be increased by paying for a higher plan, without having to physically install new hardware.
❌ Disadvantages of Cloud Storage — Exam-Quality Points
1
Requires an Internet connection — If the Internet connection is lost or slow, you cannot access or upload your files. This is a serious problem in areas with poor connectivity or during Internet outages. Local storage is always accessible regardless of Internet availability.
2
Security and privacy risks — Your data is stored on servers owned by a third-party company. There is a risk of data breaches — hackers could potentially access sensitive personal or business files. You are trusting another company with your private data.
3
Ongoing subscription cost — Most cloud services require a monthly or annual subscription fee for significant storage. Over time, this can cost more than buying a physical hard drive, especially for large amounts of data.
4
Dependency on the provider — If the cloud provider experiences downtime, goes out of business, or changes its terms, you may lose access to your data. You are dependent on a third party continuing to operate correctly.
5
Slower transfer speeds than local storage — Uploading or downloading large files is limited by your Internet speed, which is far slower than transferring data to/from a local hard drive or SSD.
✏️ 6-mark exam technique: Give 3 advantages AND 3 disadvantages, each with a developed explanation. Don't just say "you can access it anywhere" — say WHY that is useful: "…meaning you are not tied to one device and can work from any location." Each point should be 1–2 sentences to score 2 marks per point. Avoid repeating the same idea in different words.
📌 Real-world examples to use in the exam: Google Drive · Dropbox · Microsoft OneDrive · iCloud · Amazon S3. The school network home drive is a local example of file server storage (not cloud — it is on-premises).

Question 4 Medium 6 marks

Compare magnetic hard disk drives (HDDs) with solid-state drives (SSDs) as methods of secondary storage. Refer to how data is stored, speed, capacity, and durability in your answer.

✅ Mark Scheme
  • HDD uses magnetised spinning platters; north/south poles represent 1s and 0s. SSD uses NAND flash chips; trapped electrons represent 0s, empty pools represent 1s. [2]
  • SSDs are significantly faster to access data than HDDs, as there are no moving parts and no seek time/latency. [1]
  • HDDs typically offer larger capacity per £ (cheaper per GB); SSDs are more expensive per GB but capacities are increasing. [1]
  • SSDs are more durable as they have no moving parts — better for portable devices. HDDs are fragile; the read head can crash into the platter if knocked. [1]
  • HDDs are generally cheaper, making them suitable for bulk/backup storage; SSDs better for speed-critical applications (e.g. OS drive). [1]
Question 5 Hard 6 marks
A school is upgrading its computers. The IT manager is deciding between increasing the clock speed of the processors, adding more processor cores, or increasing the size of the cache memory.

Explain how each of the three factors affects CPU performance. Identify one limitation of each approach.

✅ Mark Scheme (2 marks per factor)

Clock speed:

  • Faster clock speed means more instructions processed per second, increasing performance [1]
  • Limitation: generates excessive heat; transistors have a maximum operating rate; requires expensive cooling solutions [1]

Processor cores:

  • More cores allow multiple tasks to run simultaneously (parallel processing / multitasking), improving overall performance [1]
  • Limitation: not all software can distribute tasks across multiple cores; sequential programs see no benefit from extra cores [1]

Cache size:

  • Larger cache means more frequently used data is stored close to the CPU; fewer cache misses; CPU waits less for data from RAM [1]
  • Limitation: cache is very expensive; there are physical limits to how much cache can fit on a processor chip [1]
