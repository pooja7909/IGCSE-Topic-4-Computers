🔣 Why Does Logic Matter? ▼ expand
🖥️ CPUs Use Logic

The ALU inside every CPU performs AND, OR and NOT operations billions of times per second. All conditional code (IF statements) ultimately becomes these three operations.

📊 Truth Tables are Examinable

Drawing and completing truth tables for AND, OR, NOT and combined expressions (e.g. NOT A AND B) is a core exam skill — one of the most reliably tested parts of Unit 4.

🐛 Logic Errors Cause Bugs

Logic errors are a common cause of bugs. Writing the correct logic statement before coding helps ensure the right code runs. Understanding Boolean operators is essential for all programming.

⚡ Logic Circuits

AND, OR and NOT are built directly into electronic circuits — logic gates. Every decision a computer makes, from checking a password to evaluating a game condition, uses these gates.

📐 Operator Precedence

Like BIDMAS in maths, logic has a precedence order: brackets first, then NOT, then AND, then OR. Getting this wrong in your truth table costs marks.

📝 Writing Logic Statements

You must be able to read a real-world scenario (e.g. an alarm system) and write the correct IF...THEN condition using AND, OR and NOT. This tests both logic and problem-solving skills.

✏️ Key rule: AND = both must be true. OR = at least one must be true. NOT = inverts (flips) the value. Precedence: brackets → NOT → AND → OR.

18.1Boolean Operators: AND, OR, NOT
▼

All conditions in programs — every IF statement — are ultimately calculated as Boolean logic. There are three Boolean (logical) operators: AND, OR and NOT.

SUBJECT VOCABULARY

Boolean — something that can only take the values True or False (1 or 0). Named after English mathematician George Boole.

Truth table — a table showing all possible combinations of the inputs and outputs of a logic operator or expression.

Logic circuit — an electronic circuit that has inputs and outputs that follow one of the Boolean operators (AND gate, OR gate, NOT gate).

Operator precedence — the order in which operators are applied in an expression. For logic: brackets → NOT → AND → OR.

🔵 AND Operator

The output of AND is 1 (True) only when BOTH inputs are 1. If either input is 0, the output is 0.

🎮 Example: IF health <= 0 AND lives = 0 THEN SET game_over TO true — the game only ends when BOTH conditions are true at the same time.

🟣 NOT Operator

NOT has only one input. It inverts (flips) the value — turns 1 into 0 and 0 into 1.

🎮 Example: IF NOT god_mode AND health <= 0 THEN ... — god_mode makes you invincible, so the game only ends if god_mode is OFF (NOT god_mode = 1) AND health runs out.

18.2Combined Expressions & Complex Truth Tables
▼

Real programs often combine all three operators in one condition. The key is operator precedence: evaluate in this order — brackets → NOT → AND → OR. This is the logic equivalent of BIDMAS in maths.

18.3Writing Logic Statements for Real Scenarios
▼

When writing programs, you often need to translate a real-world rule into a logic statement. Logic errors are one of the most common causes of bugs — testing your logic with a truth table before coding prevents mistakes.

🌿 Worked Example: Fertiliser Advisor

Mohammad's gardening website gives advice based on leaf colour and size:

Rule 1: Yellow leaves near soil → nitrogen fertiliser
Rule 2: Brown AND small leaves → phosphorous fertiliser
Rule 3: Brown AND normal-size leaves → potassium fertiliser
SET advice TO 'nothing needed' IF colour = 'yellow' THEN SET advice TO 'nitrogen' ELSE IF colour = 'brown' AND size = 'small' THEN SET advice TO 'phosphorous' ELSE IF colour = 'brown' AND size = 'normal' THEN SET advice TO 'potassium' END IF END IF END IF
💡 Key technique: Start by setting a default value ('nothing needed'), then only change it when conditions are met. This means you don't need to handle every possible case explicitly.

⏰ Worked Example: Alarm Clock

Nadia's alarm: 7:30am on school days, 9:00am on weekends/holidays, except 8:00am on term-time Saturdays for hockey:

SET alarm TO '7:30 am' IF term_time = false OR (day = 'Saturday' OR day = 'Sunday') THEN SET alarm TO '9:00 am' ELSE IF term_time = true AND day = 'Saturday' THEN SET alarm TO '8:00 am' END IF END IF
✏️ Exam tip — Writing logic statements: (1) Identify the variables first (what changes?). (2) Write the conditions using AND, OR, NOT. (3) Use brackets to control order. (4) Always set a default value before the conditions. The key mark is showing the correct operator (AND vs OR) for the situation described.

Question 5 Hard6 marks
A student writes this condition for a school system:
IF year = 11 AND (grade < target OR target > 7) THEN SET revision_class TO true

(a) State the operator precedence used to evaluate this expression. [2] (b) Construct a partial truth table showing: year=11, grade<target, target>7, the bracketed sub-expression, and the final output for ALL combinations where year=11. [4]

✅ Mark Scheme

(a) Precedence [2]:

  • Brackets are evaluated first: (grade < target OR target > 7) [1]
  • Then AND is evaluated: year = 11 AND (bracketed result) [1]

(b) Table (year=11 rows only) [4]:

  • Columns: year=11 | grade<target | target>7 | (grade<target OR target>7) | revision_class [1]
  • year=11=1, 0, 0 → bracket=0 → output=0 [1]
  • year=11=1, 0, 1 → bracket=1 → output=1 [1]
  • year=11=1, 1, 0 → bracket=1 → output=1 [1]
  • year=11=1, 1, 1 → bracket=1 → output=1 [1] (max 4)
Question 6 Easy3 marks

(a) What does the NOT operator do? [1] (b) How many inputs does NOT have, compared to AND and OR? [1] (c) State the correct operator precedence for evaluating: NOT A AND B OR C [1]

✅ Mark Scheme
  • (a) NOT inverts the input — turns 1 into 0 and 0 into 1 (flips the Boolean value) [1]
  • (b) NOT has ONE input; AND and OR each have TWO inputs [1]
  • (c) First: NOT A → then: (NOT A) AND B → then: result OR C. Precedence: NOT first, then AND, then OR [1]
