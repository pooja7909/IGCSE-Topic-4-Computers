💿 Why Does Software Matter? ▼ expand
🖥️ Hardware Needs Software

Hardware is useless without software. Software is what makes the hardware do something useful — from the OS managing the CPU to a word processor letting you write an essay.

⚙️ The OS is Everywhere

Every general-purpose computer runs an OS. Without it, applications couldn't communicate with the hardware at all. Understanding what the OS does is fundamental to computer science.

🛠️ Utility Software Keeps Systems Running

Anti-virus, defragmenters, backup tools and compression software are all utility programs that maintain, secure and optimise computer systems — examined regularly.

🌍 Models Change the World

Computer simulations and models let scientists test ideas without risk — from weather forecasting to flight simulators. They are powerful tools but always contain assumptions.

🔐 Security Software is Critical

Viruses, spyware and malware are real threats. Understanding what antivirus, firewalls and encryption do — and why — is increasingly important in everyday computing.

📂 File Management

Compression, defragmentation and backup are essential tools for managing disk space and data safety — all examinable utility software topics.

✏️ Key distinction: System software (OS + utilities) vs Application software. The OS sits between hardware and applications. Utility software ≠ application software — it supports the system, not the user's primary task.

19.1Types of Software: System vs Application
▼
SUBJECT VOCABULARY

Software — the set of programs run by a computer system.

Application software — software that performs a task that would otherwise be done by hand (e.g. word processor, spreadsheet, browser, games). It is the reason a user chooses to use the computer.

System software — software that supports the running of other software and the management of hardware. Includes the operating system AND utility software.

Operating system (OS) — system software designed for particular hardware which manages other programs' access to the hardware. Examples: Windows, Linux, Android, iOS.

Utility software — software that does a useful job for the user that is not essential to the OS and is not the reason for using a computer in the first place. Examples: antivirus, defragmenter, backup tools.

✏️ Exam tip: Students often confuse utility software with application software. Remember: utility software is not the reason you use the computer — you don't buy a laptop to run antivirus. It's system software that supports and maintains the system.
19.2Operating Systems — Functions in Detail
▼

A general-purpose computer needs an OS because software must work on many different hardware configurations. Rather than writing software that talks directly to hardware, programmers use the OS as a middle layer. The OS provides an API (Application Programming Interface) — a set of commands (e.g. OpenFile) that hide the complexity of the hardware.

📋 The Six Key OS Functions

1 — Hardware Management
The OS manages access to hardware devices including the CPU, RAM, and I/O devices. It uses device drivers to communicate with specific hardware — a driver translates between the OS and the physical device. This means an application can use OpenFile without knowing whether the file is on an SSD or HDD.
2 — File Management
The OS manages the directory (folder) structure, file naming, and access to secondary storage. It keeps track of where files are stored on disk, allows files to be created, moved, renamed, copied and deleted, and controls which users can access which files.
3 — Process Management & Scheduling

Each running program is a process. A von Neumann CPU can only do one thing at a time, yet multiple apps appear to run simultaneously. The OS creates this illusion using scheduling — giving each process a short slice of CPU time before switching to the next. This is called round-robin scheduling. Processes running alongside each other this way are called concurrent processes.

💡 Example: You can listen to music while typing in Word because the OS rapidly switches the CPU between the music player process and the Word process — so fast it appears simultaneous.
4 — Memory Management & Paging
When you open many applications, RAM can become full. The OS handles this using paging — it moves programs or data from RAM to the hard disk and back again as needed. This is the same as virtual memory (covered in Topic 17). A computer using paging heavily may run noticeably slower as disk access is much slower than RAM.
5 — User Interface

The OS provides the user interface (UI) — the way the user interacts with the computer. There are two main types:

GUI — Graphical User Interface

Windows, Icons, Menus, Pointer (WIMP). Examples: Windows, macOS, Android. Easy to learn, no commands to memorise, visual feedback. Used by most general-purpose computers.

CLI — Command-Line Interface

Text commands typed by the user (e.g. MS-DOS, Linux terminal). Faster for experts, uses less memory, more precise control. Used by servers and technical users who prefer speed over ease.

6 — Authentication & Security

Authentication is the process of proving to a computer system who you are (e.g. using a username and password, fingerprint, or PIN). In multi-user systems (like a school network), each user must authenticate. Authority is based on identity — learner users may not be able to print, while tutor users can. There is usually a super-user (administrator/root) who can access all resources and control others' access rights.

✏️ Exam tip: A common question is "state four functions of an operating system." Use the six above — pick any four and state each clearly: file management, process scheduling, memory management/paging, user interface, authentication, hardware management. One function per mark — don't write paragraphs for a "state" question.
19.3Utility Software — Types & Functions
▼

Utility software does a useful job for the user but is not essential to the OS and is not the primary reason for buying a computer. It falls into three areas: basic tools, file management tools, and security tools.

🛠️ File Management Utilities

💾 Backup Software
What it does: Makes a copy of files in another location so they are still available if the original is lost or damaged.

Why it's needed: Hardware can fail, files can be accidentally deleted, ransomware can encrypt files. Regular backups prevent data loss.

✅ Protects against data loss · Can recover deleted files
📦 Compression Software
What it does: Reduces file size by encoding data more efficiently (e.g. ZIP files). Frees up disk space.

Trade-off: Compressed files are slower to access — they must be decompressed before use.

✅ Saves disk space · Faster to upload/email
❌ Slower to open · ⚠ NOT the same as encryption!
🔀 Defragmenter
What it does: Files on a hard disk are stored in chunks called clusters. A single file's clusters can be spread across the disk (fragmented). The defragmenter moves clusters so they are physically closer together, speeding up disk access.

⚠ Only for HDDs! SSDs do not need defragmenting — access times are the same regardless of where data is stored.
🔁 File Converter
What it does: Converts files between different formats. Especially useful for multimedia files (images, video, audio) which have many common formats (jpg, png, gif, mp3, mp4, etc.).

Example: Converting a .WAV audio file to .MP3 reduces the file size significantly.

🔐 Security Utilities

🦠 Anti-virus Software
Detects and removes viruses and malware. A virus copies itself and spreads — it can steal data, send spam, crash servers, or simply annoy users. Antivirus identifies code that copies itself and prevents those processes from running.
🕵️ Anti-spyware
Spyware is software installed secretly that records private information as the user enters it (e.g. keystrokes, passwords, bank details). Anti-spyware detects and removes these hidden programs.
🔥 Firewall
Controls program access to the network — both incoming and outgoing traffic. Decides which data is allowed to flow to and from the computer. Can prevent infection by viruses and stop malware from sending data out.
🔐 Encryption software makes files unreadable without a password. Data on a laptop might be encrypted so it is not accessible if the laptop is stolen — even if the hard drive is removed.
✏️ Exam tip: Know the difference between a virus (self-copies, spreads) and spyware (secretly records data). Both are malware but work differently. A firewall controls NETWORK traffic — it is not the same as antivirus.
19.4Simulation & Computer Modelling
▼

A computer model is a program that simulates a real-world system or process. It uses abstraction — hiding complexity and only modelling the key features. It allows scientists, engineers and economists to ask 'what if?' questions without interfering with the real world.

SUBJECT VOCABULARY

'What if?' question — running a computer model with a given set of inputs to see what the model produces as an output or prediction. E.g. "What if the bridge were twice as long?"

Heuristic — a type of algorithm that finds a good (not necessarily perfect) solution quickly through trial and error and educated guesswork. Useful when exact solutions are too slow or impossible to calculate.

Monte Carlo method — carrying out a large number of random samples to get approximate solutions. The more samples used, the more accurate the result.

Neural networks — systems that process information in a way similar to human brains, learning and adapting over time. Useful for pattern recognition (faces, medical diagnosis, etc.).

✈️ Real-World Examples

✈️ Flight Simulator

Pilots train to fly safely without endangering real passengers. Uses real controls and a moveable cabin to feel realistic. Mistakes are safe — no lives at risk.

🌦 Weather Forecasting

Atmospheric models split the atmosphere into sections, tracking temperature, pressure and humidity. Each section's effect on neighbours is calculated to predict weather patterns.

🌍 Earthquake Prediction

Uses heuristics, Monte Carlo methods and neural networks to predict earthquake patterns based on probability and past data — since the exact maths behind earthquakes is unknown.

⚖️ Advantages & Limitations of Computer Models

✅ Advantages
✓
Test scenarios that are impossible, dangerous or unethical in the real world
✓
Ask 'what if?' questions quickly and cheaply
✓
Can be run many times with different inputs
✓
Reveal patterns in complex systems that humans couldn't calculate
❌ Limitations
✗
Contains assumptions — it is not reality, so answers may not be correct
✗
The real world is too complex to model every factor — abstraction means detail is lost
✗
Results are predictions — they should be treated with caution, not as certainty
✗
Only as good as the data and rules used to build the model
✏️ Exam tip: For "give two advantages of using a computer model" — link each to a real reason: e.g. "Pilots can make mistakes in a flight simulator without risk to passengers" (not just "it is safer"). For limitations, always mention "assumptions" and "simplification" — these are the key marking points.

Question 3 Medium4 marks

Describe what a defragmenter does and explain why it is only useful for hard disk drives (HDDs) and not solid-state drives (SSDs).

✅ Mark Scheme
  • Files are stored on a hard disk in chunks called clusters [1]
  • A fragmented file has its clusters spread across the disk — the read head must move many times to access it, which is slow [1]
  • A defragmenter moves clusters so that all the parts of each file are stored closer together / contiguously, speeding up disk access [1]
  • SSDs have no moving parts — access time is the same regardless of where data is stored on the chip, so defragmentation provides no speed benefit [1]

Question 8 Hard5 marks
A user has many programs open on their computer at the same time. The computer is running slowly.

Explain two reasons why the computer might be running slowly. For each, name the OS mechanism involved and describe what the OS does to manage the situation.

✅ Mark Scheme

Reason 1 — CPU Scheduling:

  • Many processes are competing for CPU time. The OS uses scheduling to give each process a short time-slice in turn, creating the illusion they all run simultaneously. With many processes, each gets less CPU time → slower. [up to 2]

Reason 2 — Paging / Virtual Memory:

  • RAM may be full with many programs loaded. The OS uses paging — moving programs from RAM to the hard disk swap area and back when needed. Hard disk access is much slower than RAM, causing a significant drop in performance / disk thrashing. [up to 3]
