🖥 Why Do Machines and Computational Models Matter? ▼ expand
🏭 Embedded Everywhere

Every modern device from a car to a washing machine to a microwave contains an embedded computer following the IPO model. Understanding this is fundamental.

⚙️ How All Computers Work

The input–process–output model explains how EVERY computer operates — regardless of its size, shape or purpose. It is the foundation of computer science.

⚡ Faster Processing

Parallel models allow computers to solve problems much faster by using multiple processors simultaneously — essential for weather forecasting, AI, and rendering.

🤖 Artificial Intelligence

Multi-agent models form the basis of AI. Autonomous agents that negotiate and cooperate are the building blocks of smart assistants, self-driving cars, and more.

📐 Algorithm Design

Understanding processing models helps programmers write better algorithms — knowing whether to use sequential, parallel or agent-based approaches for different problems.

🌍 Real-World Applications

From smart home systems to online booking agents — multi-agent and parallel models are behind the most powerful computing applications in the world today.

✏️ Key distinction: Sequential ≠ Parallel ≠ Multi-agent. Know the difference: sequential = one processor, one step at a time. Parallel = one task, split across processors. Multi-agent = separate autonomous agents handling separate tasks.

🖥 Why Does the IPO Model & Computational Models Matter? ▼ expand
⚙️ Everything is a Computer

From washing machines to cars to smartphones — they all follow the IPO model. Recognising this is key to understanding modern technology.

📐 Algorithm = The Rules

The processing step is where an algorithm runs. A good algorithm is what separates a useful computer from a useless one — it's the "clever bit".

🚀 Why Parallel Matters

Modern computers have multiple cores — they use parallel processing every second you use them. Without it, complex tasks like gaming or video editing would be impossibly slow.

🤖 The Future: AI Agents

Multi-agent systems underpin AI assistants, recommendation algorithms, autonomous vehicles, and smart cities — the defining technologies of the 21st century.

✏️ Exam distinction: Parallel = same computer, one task split. Multi-agent = different systems, different autonomous tasks. Never confuse these two!
16.1The Input–Process–Output Model
▼

You might be surprised to know that every device — a car, a printer, a washing machine, a smartphone and a remote control — all contain an embedded computer. A computer is a machine that takes some kind of input from its surroundings, processes the input according to given rules, and provides some kind of output.

SUBJECT VOCABULARY

Input — to enter data into a computer. The raw data a computer receives from the outside world via input devices or sensors.

Process — to change the meaning or format of some data. The CPU runs an algorithm on the input to transform it.

Output — to display or output data that has been processed (or has been stored). The result of processing, sent to an output device.

Embedded computer — a computer fixed firmly into another object (e.g. washing machine, car). Still follows the IPO model.

🏭 Real Device Examples

🖥 Desktop Computer
Input: Keyboard (typing), mouse (clicking), webcam, microphone
Process: CPU runs programs — word processing, calculations, games
Output: Monitor, speakers, printer
🧺 Washing Machine
Input: Control panel settings (temperature, spin speed), water level sensor, door sensor
Process: Embedded computer calculates motor speed, water heat, cycle time
Output: Motor spinning drum, heater heating water, pump draining water
📱 Smartphone
Input: Touchscreen, microphone, camera, GPS, accelerometer
Process: Processor runs apps, handles calls, compresses photos
Output: Screen, speaker, vibration motor, wireless signals
🚗 Car (Engine Management)
Input: Fuel sensors, temperature sensors, speed sensors, ignition
Process: ECU calculates optimal fuel injection and ignition timing
Output: Fuel injectors, ignition system, dashboard warning lights

🔧 Adding Storage to the Model

You are familiar with a desktop computer: the keyboard provides input and the screen provides output. In a similar way, the embedded computer in a washing machine has controls that provide input and a motor and heater as outputs.

The clever bit in computer science is what goes on between the inputs and outputs — the processing. Processing means performing a series of actions on the inputs, according to a given set of rules. This set of rules is called an algorithm.

If you try to follow a more complicated algorithm, you will probably need to keep track of your working. In a similar way, anything beyond the simplest algorithm needs temporary storage or memory. We add storage to the IPO diagram:

DEFINITION

Computational model — describes how algorithms are executed by a machine, e.g. by processing data using inputs and outputs. There are different models of how this processing can be carried out.

In the sequential model, this is the way that we commonly imagine how computers work. This involves following instructions in an algorithm step by step, in order, from start to finish.

16.3Parallel Computational Model
▼

In the parallel model, computer processes are distributed between two or more processors in a computer with two or more processors installed. Each separate part of the algorithm that each processor processes can be combined together.

⚡ What it requires:
1
An operating system capable of supporting two or more processors
2
Software programs capable of distributing processes between them equally
⚙️ How it works:
1
A large task is split into parts
2
Each part given to a different processor
3
All processors work simultaneously
4
Results are combined at the end
🎨 Example: Rendering a 3D animation frame — the image is divided into sections. Processor 1 renders top-left, Processor 2 renders top-right, etc. — all at the same time. The sections are combined into the final image. Much faster than one processor rendering the whole frame sequentially.
✏️ Exam tip — Parallel vs Multi-Agent: In parallel, it is ONE task split across processors in the same computer. In multi-agent, it is different tasks given to different independent systems. This is the most common exam confusion — learn the difference carefully!
16.4Multi-Agent Computational Model
▼

In the multi-agent model, separate tasks or algorithms are processed by different systems (agents) to perform a particular function. Each agent acts independently and under its own control. Each agent is autonomous, but they cooperate with each other through negotiation and coordination.

This is the basis of artificial intelligence and reflects our human society, where independent people interact, negotiate and cooperate to maintain our society.

SUBJECT VOCABULARY

Agent — a computer system that can interpret its environment (through its sensors). It can act autonomously upon that environment (through its effectors).

Autonomous — self-directed and acts without requiring any help or guidance. Can choose its own goal and uses its experience to achieve it.

Negotiation — a discussion in order to reach an agreement (between agents).

Coordination — organising different activities so that they all work together effectively.

Effector — something that produces an effect by carrying out an action.

🏖️ The Holiday Booking Example (from the textbook)

Searching the Internet for the answer to a specific query can be a long and boring process that could be done for us by an agent — a computer program that would give the query details of what we needed to find out.

If we were organising a holiday, we could communicate our preferences and different agents could cooperate to search and compare different deals, negotiate for the best prices, plan transport and book airlines. They could even order sun tan lotion or book restaurants for us without our direct involvement!

MED Question 7
[4 marks]
A software company is rendering a 3D animated film. Each frame of the film is very complex and takes a long time to process.
Explain how the parallel computational model could be used to speed up the rendering of the film. [4 marks]
✅ Mark Scheme (any 4):
  • Each frame / the rendering task is split into smaller parts (1)
  • Each part is given to a different processor (1)
  • All processors work on their part simultaneously / at the same time (1)
  • The results from each processor are combined into the final frame (1)
  • This is faster than a single processor doing all the work sequentially (1)
  • Requires an OS supporting multiple processors AND software to distribute the task equally (1)
MED Question 8
[4 marks]
State what is meant by 'autonomous' and explain the role of negotiation and coordination in a multi-agent system. [4 marks]
✅ Mark Scheme:
  • Autonomous: self-directed / acts without requiring help or guidance / can choose its own goal and determine how to achieve it (1)
  • Negotiation: a discussion between agents in order to reach an agreement, e.g. one agent proposes a price and another accepts or counters it (1)
  • Coordination: organising the different activities of agents so that they all work together effectively towards the shared goal (1)
  • Together, negotiation and coordination allow agents that are acting independently to still cooperate and produce a useful combined result (1)
MED Question 9
[4 marks]
Explain the difference between the parallel and multi-agent computational models. [4 marks]
✅ Mark Scheme:
  • In the parallel model, one task is split into parts and processed by multiple processors in the same computer (1)
  • The parts are combined to produce a single result (1)
  • In the multi-agent model, separate/different tasks are processed by different independent systems (agents), which may be on entirely different computers (1)
  • Each agent acts autonomously under its own control, cooperating through negotiation and coordination — rather than being controlled centrally (1)

[6 marks]
A travel company wants to build a system that allows customers to plan a complete holiday. The customer provides their preferences (destination, dates, budget). The system must: find available flights, search for hotels, compare prices, negotiate deals and make all bookings automatically — without the customer needing to do anything else.
Explain how the multi-agent computational model could be used to implement this system. In your answer, refer to the terms: agent, autonomous, negotiation, coordination, effector. [6 marks]
✅ Mark Scheme — any 6 from:
  • The system uses multiple agents — separate computer systems each assigned a different task (1)
  • Each agent is autonomous — acts independently, under its own control, without needing human instruction at each step (1)
  • Agent 1 searches and compares all available flights for the given dates / destination (1)
  • Agent 2 searches for hotels matching the customer's preferences and budget (1)
  • A pricing / comparison agent uses negotiation — discussing prices with flight and hotel agents to reach the best combined deal (1)
  • Coordination ensures all agents work together effectively — e.g. the flight and hotel dates are aligned and the total stays within budget (1)
  • Once the best deal is agreed, a booking agent uses its effectors to act on the environment — sending booking confirmations, making payments, emailing the customer (1)
  • This reflects the multi-agent model: separate tasks, independent agents, cooperating to produce a complete holiday plan (1)
HARD Question 11
[6 marks]
Compare the three computational models — sequential, parallel and multi-agent — in terms of how they process tasks, how many processors/systems are involved, and an example application for each. [6 marks]
✅ Mark Scheme — 2 marks per model (description + example):
  • Sequential: instructions executed one at a time in order by a single processor; each step completes before the next begins (1); e.g. a simple calculator program, a recipe-following algorithm (1)
  • Parallel: one task split into parts, each part processed simultaneously by different processors in the same computer, results combined (1); e.g. rendering a 3D animation, weather forecasting simulation (1)
  • Multi-agent: separate tasks handled by different autonomous independent agents that cooperate through negotiation and coordination (1); e.g. AI holiday booking assistant, self-driving car systems, smart home automation (1)
